Building StandClock with Rust

A journey of building a desktop app

My Problem

I was sitting 10+ hours a day at my desk

  • Back pain getting worse every week
  • I'd get so focused on coding, I'd forget to move
  • Tried phone apps, browser timers - nothing stuck
  • Needed something that would just... be there

So I decided to build my own solution

The Idea: StandClock

A simple desktop app that reminds me to take breaks

  • Pomodoro-style timer: 25 min work, 5 min break
  • Choose your break activity: Stretch, walk, water
  • Track what works: See patterns over time
  • Lightweight: Won't slow down my computer
  • Always running: No need to remember to start it

Why Not Just a Website?

I tried web timers first. Here's what happened:

  • Closed by accident: One wrong click and my timer is gone
  • No real notifications: Browser tabs can't interrupt me properly
  • Only works in browser: When browser is closed, no reminders
  • Data scattered: Different devices, different history

I needed a real desktop app

The Big Question

How do I build a desktop app?

I'm a web developer. I know HTML, CSS, JavaScript.

But desktop apps? That's a different world...

Two Options for Building Desktop Apps

Electron

Popular, mature

VS Code, Slack, Discord

~150 MB apps

Tauri

New, fast, small

Built with Rust

~6 MB apps

Why I Chose Tauri

This app runs all day in the background

  • Tauri apps are 15x smaller (6 MB vs 150 MB)
  • Uses 3x less memory (won't slow down your computer)
  • Starts instantly (under 1 second)
  • Better battery life (important for laptops)

Perfect for a background timer app

What Is Tauri?

Web frontend + Rust backend = Tiny desktop app

  • Build UI with HTML/CSS/JavaScript
  • Use Rust for system operations
  • Uses your OS's browser (no need to bundle one)

The tradeoff: I had to learn Rust
(But it was worth it)

So I Started Building

What I needed was simple:

  • Timer that counts down
  • Pick my break activity
  • Notification when time's up
  • Track my history

Simple, right?

Attempt 1: Go Full Rust

My ambitious plan: Use Rust for everything

  • Frontend: Sycamore (Rust framework with WASM)
  • Backend: Rust (obviously)
  • Goal: Pure Rust, no JavaScript

This failed spectacularly

Why It Failed

  • Sycamore was too new: Almost no documentation or examples
  • Everything took forever: Simple UI changes = hours of work
  • No component libraries: Had to build everything from scratch
  • Debugging was painful: Error messages were cryptic

After 2 weeks, I had a button and a timer. That's it.

Time to pivot

Attempt 2: Hybrid Approach

New plan: Use the right tool for each job

  • Frontend: Solid.js (familiar, fast, good docs)
  • Backend: Rust (for system operations)
  • UI Library: DaisyUI (32 themes for free!)
  • Database: SQLite (simple, local)

This worked beautifully

The Final Tech Stack

  • UI: Solid.js + DaisyUI (32 themes!)
  • Backend: Rust + Tauri
  • Database: SQLite (stores history locally)

Result: 6 MB app that uses 50 MB RAM

What I Learned

The Hard Parts:

  • Learning Rust took time
  • Smaller community = fewer examples online
  • First attempt (all Rust) failed after 2 weeks

The Wins:

  • 6 MB app, 50 MB RAM, instant startup
  • Once I learned Rust basics, development was fast
  • Type safety caught bugs before testing

Should You Use Tauri?

Use Tauri if:

  • Background apps, utility tools
  • Performance matters
  • Want to learn Rust

Use Electron if:

  • Need to ship fast
  • JavaScript-only team
  • Complex apps with lots of Node.js libraries
